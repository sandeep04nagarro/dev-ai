# Documentation Skill

## Overview
This skill guides the agent in creating high-quality documentation, including READMEs, API references, and internal design documents.

## Instructions
### 1. Identify the Audience
- Tailor the language and level of detail to the target audience (e.g., users, developers, contributors).

### 2. Structure the Content
- Use clear headings and subheadings.
- Provide a logical flow from overview to specific details.
- Include a Table of Contents for long documents.

### 3. Write Clearly and Concisely
- Use active voice and simple language.
- Avoid jargon unless it's defined or widely understood.
- Use bullet points and numbered lists for readability.

### 4. Use Examples
- Provide code snippets, configuration examples, or diagrams to illustrate concepts.
- Ensure examples are correct and easy to follow.

### 5. Keep it Updated
- Ensure documentation matches the current state of the code.
- Link to relevant external resources where appropriate.
