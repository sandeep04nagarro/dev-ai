# Testing Skill

## Overview
This skill helps the agent design and implement effective tests. It covers unit testing, integration testing, and automated validation strategies.

## Instructions
### 1. Identify Test Scenarios
- Determine the core functionality to be tested.
- Identify edge cases, error conditions, and boundary values.
- Consider both positive and negative testing.

### 2. Design Test Cases
- Ensure tests are independent and repeatable.
- Use descriptive names for test cases.
- Follow the Arrange-Act-Assert pattern.

### 3. Implement Tests
- Use the project's preferred testing framework (e.g., pytest).
- Mock external dependencies where appropriate to isolate the unit under test.
- Ensure tests are efficient and don't introduce unnecessary delays.

### 4. Verify Coverage
- Aim for high branch and statement coverage.
- Prioritize testing critical paths and complex logic.

### 5. Maintain Tests
- Update tests when functionality changes.
- Refactor tests for clarity and maintainability.
